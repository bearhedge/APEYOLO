Build a clean, monochrome (black/white/greys) web app for Apex Options using Orbitron as the only font (import from Google Fonts; apply globally). No neon, no animated backgrounds, no gradients—minimal, futuristic UI.

Goal: an end-to-end, agentic options seller MVP with simple onboarding → automated trading → clear oversight.

Information architecture & flow (routes):
	1.	Home (/): short value prop, “Get Started” → Onboarding.
	2.	Onboarding (/onboarding): stepper with (a) Login with Google (mock OK), (b) Connect IBKR (button triggers server call to OAuth2+SSO; show ✅ when /api/broker/diag returns oauth=200 & sso=200), (c) Risk Preferences (aggression slider 0–100, leverage cap, max loss/day, per-symbol cap). “Finish” → Dashboard.
	3.	Dashboard (/dashboard, rename from Positions): cards in a responsive grid:
• Connection (Google + IBKR status, “Test Connection” → /api/account).
• Agent Control (Strategy selector: CSP/CC, symbols input, Start → POST /api/agent/start, Stop → POST /api/agent/stop, live status from GET /api/agent/status).
• Risk Snapshot (read-only from saved prefs).
• Account Overview (NAV, buying power, margin used %, open positions).
• Open Positions table (symbol, side, qty, strike, exp, avg, mark, P/L$, P/L%).
	4.	PNL (Track Record) (/pnl): immutable history table (paginated + CSV export): ts, symbol, side, qty, strike, exp, fillPrice, fees, realizedPnL, unrealizedPnL, netPnL, orderId, algoTag. Compute and display rowHash = SHA256(JSON(row)) (first 8 chars) and a footer datasetHash (SHA256 of all rowHashes). Add a disabled “Anchor to chain” button (placeholder).
	5.	Settings (/settings):
• IBKR Connection card: show clientId, credential (read-only), last diag + traceId, “Reconnect” (re-runs OAuth2+SSO flow).
• Risk Controls: editable aggression, leverage cap, max loss/day, per-symbol exposure (persist locally).
• Safety Rails: toggles for “halt on drawdown %”, “max trades/day”, “market hours only”.
• Notifications: email/webhook fields (store only).
• Agent Defaults: default strategy and allowed symbols.

Implementation notes:
	•	Use a consistent component library/look (cards, pills, inputs) in Orbitron; strictly monochrome accents.
	•	Central nav with routes: Home, Dashboard, PNL, Settings.
	•	API layer (src/lib/api.ts) with typed functions; return mocks if backend 404s:
getDiag(), getAccount(), getAgentStatus(), startAgent(payload), stopAgent(), getPositions(), getPNL().
	•	Minimal state store (Zustand or Context):
auth{googleConnected}, broker{ibkrConnected,lastDiag}, agent{status:'stopped'|'running'|'error'}, risk{aggression,maxLeverage,maxDailyLoss,maxPerSymbol} (persist).
	•	Hash util (browser Web Crypto) for PNL row/dataset hashes.
	•	Keep typography/layout consistent; no color noise; responsive grid.

Deliver a working scaffold with these pages, routes, cards, tables, API stubs, and state—ready to swap mocks for live IBKR endpoints later.
